# Sovereignty v2.1 Roadmap (proposed)

## Status

Decision-pending. v2.0.0 ships first (proof v2 + production hardening); v2.1 picks up the App UI work and transport/engine generalizations deferred from the v2.0.0 dogfood swarm.

## Open questions for v2.1 kickoff

### App UI

1. **Framework** — Tauri (Rust shell + web UI, smallest binary), Textual (terminal-native, matches CLI ethos), Electron (heaviest, web-dev friendly), or PyQt (native Python, no JS)?
2. **Scope** — full game UI (board + cards + actions), audit viewer for XRPL-anchored proofs, or both?
3. **Distribution** — bundle with PyPI, separate npm distribution like the current PyInstaller binaries, or standalone Tauri/Electron app?
4. **Design tooling** — Claude artifacts/preview for mockups, or a specific MCP we identify together?

### Transport / engine generalizations (deferred from v2.0.0 Feature Audit)

- LedgerTransport ABC ledger-generalization (currently XRPL-shaped)
- Async transport sibling
- Mainnet anchor support (currently testnet-only)
- Multi-tx anchor consolidation
- Daemon / web-service mode
- `sov undo` (state-history journal — multi-day refactor)
- `sov resume <game-id>` (multi-save model)

## Decision

Post-v2.0.0 ship. Either a separate dogfood swarm or a focused initiative; framework choice is the first review gate.
