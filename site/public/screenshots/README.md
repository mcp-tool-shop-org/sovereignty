# Sovereignty v2.1 desktop-app screenshots

This directory holds the three screenshots embedded in the root `README.md`
"Desktop app" section and referenced from the landing page (`site/`).

The PNG files are **not** generated by the docs agent — they require a graphical
environment (running Tauri shell + screen capture) that the autonomous swarm
agent does not have. Mike runs the recipe below and commits the binaries.

## Targets

Save each capture as PNG at exactly **1280×800** (full window, including the
macOS title bar). Dark mode only (v2.1 ships dark only — no light theme).

| Path | View | Demo state |
|---|---|---|
| `audit-viewer.png` | `/audit` route | Games-list expanded with one game showing per-round verify status (mix of ANCHORED + PENDING rounds visible). |
| `game-shell.png` | `/game` route | Active game mid-round: player resource cards rendered, round timeline shows 3 prior rounds, last-N SSE event log populated. |
| `settings.png` | `/settings` route | Daemon network selector visible (testnet / mainnet / devnet), daemon connection status row populated. |

## Locked decisions (Wave 13 Stage 9-D triage Q4)

- **Capture mode:** full window (title bar + content), 1280×800. Matches the
  default Tauri window size in `app/src-tauri/tauri.conf.json`.
- **OS:** macOS canonical (matches Mike's primary dev rig).
- **Theme:** dark mode only.
- **Demo state:** `sov play campfire_v1` + 3 rounds of mock interactions.

## Recipe

```bash
# 1. Build / run the Tauri shell
cd /Volumes/T9-Shared/AI/sovereignty
pip install -e '.[xrpl,daemon]'
cd app && npm install && cd ..
cargo build --manifest-path app/src-tauri/Cargo.toml

# 2. Populate demo state in a clean scratch dir
mkdir -p ~/dev-scratch/sov-screenshots && cd ~/dev-scratch/sov-screenshots
sov play campfire_v1
# Play 3 rounds (use sov tutorial or scripted sov turn / sov end-round).
# At least one round must be anchored to populate the Audit Viewer:
sov anchor --network testnet

# 3. Start the dev shell (auto-starts daemon in readonly mode)
cd /Volumes/T9-Shared/AI/sovereignty
npm --prefix app run tauri dev

# 4. Capture each view at 1280×800 full window (dark mode), macOS canonical.
#    Save to:
#      site/public/screenshots/audit-viewer.png
#      site/public/screenshots/game-shell.png
#      site/public/screenshots/settings.png
#    macOS Cmd+Shift+4, then Space, then click the Tauri window — captures
#    full window with shadow. Crop / resize to 1280×800 if needed.

# 5. Verify dimensions
file site/public/screenshots/*.png
# All three should report: PNG image data, 1280 x 800
```

## Why recipe-only at Wave 13

The Wave 13 docs agent is autonomous and has no graphical environment, so
generating PNG binaries from a running Tauri shell is not possible inside the
swarm phase. The README embed + landing-page reference + hero badge bump all
landed; the PNGs are the only deliverable that crosses the human-in-the-loop
boundary.

Once Mike runs the recipe and commits the three PNGs, no further docs changes
are needed — the embed paths in `README.md` and the references in
`site/src/site-config.ts` already point at the final filenames.
