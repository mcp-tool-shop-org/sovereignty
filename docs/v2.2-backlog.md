## Health A — Test-warning hygiene

These are test-environment artifacts (not behavior failures). Worth a single coordinated cleanup pass during v2.2 Health A, after the v2.1.0 release has stabilized and any regressions surface.

- **`PanicModal.test.tsx` `act()` warning on dismiss-button test.** Wave 10 coordinator subset (2026-05-02). The dismiss-button test fires a state update that's not wrapped in `act(...)`. Behavior is correct — modal renders + dismisses — but React emits a warning that pollutes the vitest output. Fix: wrap the dismiss interaction in `act(...)` per the React testing docs link in the warning. Single-test, ~3 LOC.
- **SSE-test skip lift.** `tests/test_daemon_sse.py` had a fixture-buffering issue with `httpx.ASGITransport` 0.28; Stage 7-B switched to a real uvicorn fixture + pytest-httpserver. The tests now pass under the curated `-W error::DeprecationWarning` + websockets-ignore filter chain. If httpx ASGITransport semantics change in a future release (the buffering behavior was the root cause), the fixture pattern can simplify back to ASGITransport — at which point the websockets/uvicorn `-W ignore` filters can be tightened too. Not a code fix; a periodic dependency-drift review.

Pairs naturally: both are test-environment hygiene, both bounded, both would land in a single wave's amend.
