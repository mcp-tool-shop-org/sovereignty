## 2. IPC topology

**Webview talks directly to daemon over HTTP/SSE on `127.0.0.1:<port>` with bearer token.** Rust shell does NOT proxy daemon calls.

Why: the daemon is already an IPC server (Wave 3 shipped it). Rust shell-as-proxy adds a hop with no value. Tauri shell becomes a thin window manager + 4-command bridge for daemon discovery/lifecycle.

```
┌─────────────────────────────────────────┐
│ Tauri 2 Window                          │
│  ┌──────────────────────────────────┐   │
│  │ Webview (React + TS)             │   │
│  │   fetch('http://127.0.0.1:PORT') │   │
│  │   new EventSource('.../events')  │   │
│  └──────────┬───────────────────────┘   │
│             │ Tauri invoke              │
│             ▼                           │
│  ┌──────────────────────────────────┐   │
│  │ Rust shell (4 commands only)     │   │
│  │   daemon_status / start / stop   │   │
│  │   get_daemon_config              │   │
│  └──────────┬───────────────────────┘   │
└─────────────┼───────────────────────────┘
              │ subprocess `sov` / fs read
              ▼
        ┌──────────────┐         ┌──────────────────┐
        │ sov daemon   │  HTTP   │ sov daemon       │
        │ {start,stop, │◄────────│ (running process)│
        │  status}     │         │                  │
        └──────────────┘         └──────────────────┘
```

## 3. Tauri command surface (4 commands)

Rust shell shells out to `sov daemon {start,stop,status}` for lifecycle (lifecycle logic stays in Python where Wave 3 tested it). `get_daemon_config` reads `.sov/daemon.json` directly (no subprocess; file is the source of truth).

| Command | Args | Returns | Implementation |
|---|---|---|---|
| `daemon_status()` | — | `DaemonStatus { state: "running"\|"stale"\|"none", config?: DaemonConfig, started_by_shell: bool }` | Runs `sov daemon status --json`, parses. `started_by_shell` reflects the shell's `AtomicBool` (true iff THIS shell instance launched the daemon via `daemon_start`); used by Settings network switcher to enforce Guardrail #1 (refuse switch on externally-started daemon). Added Wave 6 Stage A (was silently bypassed pre-Stage-A via `?? true` fallback). |
| `daemon_start(readonly: bool, network: Option<String>)` | readonly, network | `Result<DaemonConfig, ShellError>` | Runs `sov daemon start [--readonly] [--network X]`, parses, sets `started_by_shell=true` flag. |
| `daemon_stop()` | — | `Result<(), ShellError>` | Runs `sov daemon stop`. Idempotent — succeeds even if already stopped. |
| `get_daemon_config()` | — | `Result<DaemonConfig, ShellError>` | Reads `.sov/daemon.json` directly (no subprocess). Errors if file absent. |

`DaemonConfig`:
```rust
struct DaemonConfig {
    pid: u32,
    port: u16,
    token: String,
    network: String,
    readonly: bool,
    ipc_version: u32,
    started_iso: String,
}
```

`ShellError` is a tagged enum with stable `code` field for frontend dispatch:
```rust
enum ShellError {
    DaemonNotRunning,
    DaemonStartFailed { stderr: String },
    DaemonNotInstalled,           // sov daemon --help fails (no [daemon] extra)
    ConfigFileMissing,
    ConfigFileMalformed,
    SubprocessFailed { code: i32, stderr: String },
}
```

Serializes as `{"code": "DaemonNotRunning"}` etc. for typed frontend handling.

`ShellError` is mirrored to TS at `app/src/types/daemon.ts` (Wave 7 Stage B added the mirror + sync test) so frontend dispatch stays type-safe across the shell boundary.

## 3.1 Doctor coverage — `[daemon]` extra parity (Wave 7 Stage B)

The Tauri shell hard-depends on the Python `[daemon]` extra at runtime — it shells out to `sov daemon {start, stop, status}`. Operators can install the shell binary without the extra (e.g. `pip install sovereignty-game` without `[daemon]`), hitting a deep `DaemonNotInstalled` error only when they click "Start daemon."

`sov doctor` adds a `tauri_shell_extra_parity` check to surface the gap up front:

| State | Result |
|---|---|
| No Tauri shell installed (no `app/src-tauri/target/release/` artifact AND no Tauri-bundled binary on PATH) | n/a — skip |
| Tauri shell installed AND `[daemon]` extra installed (`import sov_daemon.server` succeeds) | ok |
| Tauri shell installed AND `[daemon]` extra missing | warn — hint: `pip install 'sovereignty-game[daemon]'` |

Detection of "Tauri shell installed" is a pure-fs check (search platform-conventional install locations); MUST NOT spawn the binary. Doctor wall-time <2s preserved.

## 4. Daemon lifecycle from shell

**Shell startup sequence:**

1. On window mount, frontend calls `invoke('daemon_status')`.
2. If `state === "running"`: attach. `started_by_shell = false` (in Rust shell state, `AtomicBool` with `Ordering::SeqCst`).
3. If `state === "none"` or `"stale"`: frontend calls `invoke('daemon_start', { readonly: true })`. On success, `started_by_shell = true`.
4. Frontend calls `invoke('get_daemon_config')` to receive URL + token, stores in `useDaemon` context.
5. Frontend opens `EventSource(`${url}/events`)` with token via `useDaemonEvents` hook.

**Shell shutdown sequence:**

1. Tauri `WindowEvent::CloseRequested` handler fires.
2. If `started_by_shell == true`: invoke `daemon_stop()`. Wait up to 10s.
3. Window closes. (Externally-started daemons stay alive — user controls them via CLI.)

**Why shell-side flag, not daemon-side:**

`started_by_shell` is shell-process-lifetime state, never persisted. Daemon trust boundary stays clean (daemon doesn't know or care who started it). If shell crashes, flag dies with it, daemon stays alive — user can `sov daemon stop` manually. Robust failure mode.

**Default spawn mode: `--readonly`.** Audit viewer (Wave 5) doesn't need anchor capability. Game shell (Wave 5) decision deferred — if it needs anchor, Wave 5 advisor call adds full-mode toggle. v2.1 ships readonly default.

## 7. Mechanical type-sync test

Lives in `tests/test_daemon_types_ts_in_sync.py` (docs domain — pins doc↔code consistency).

```python
"""Pin doc↔code consistency: SSE events + error codes from
docs/v2.1-daemon-ipc.md must appear as TS string literals in
app/src/types/daemon.ts. Drift fails CI."""

from pathlib import Path
import pytest

REPO_ROOT = Path(__file__).resolve().parent.parent
TYPES_TS = REPO_ROOT / "app" / "src" / "types" / "daemon.ts"

SSE_EVENTS = {
    "daemon.ready",
    "daemon.shutdown",
    "anchor.pending_added",
    "anchor.batch_complete",
    "game.state_changed",
    "error",
}

DAEMON_ERROR_CODES = {
    "DAEMON_READONLY",
    "DAEMON_AUTH_MISSING",
    "DAEMON_AUTH_INVALID",
    "DAEMON_PORT_BUSY",
    "DAEMON_NOT_INSTALLED",
    "MAINNET_FAUCET_REJECTED",
    "ANCHOR_PENDING",
}

@pytest.mark.parametrize("literal", sorted(SSE_EVENTS | DAEMON_ERROR_CODES))
def test_daemon_ts_contains_literal(literal: str) -> None:
    """Every SSE event type and daemon error code must appear as a
    string literal in app/src/types/daemon.ts."""
    if not TYPES_TS.exists():
        pytest.skip("app/src/types/daemon.ts not present (pre-Wave-4)")
    text = TYPES_TS.read_text(encoding="utf-8")
    found = (f'"{literal}"' in text) or (f"'{literal}'" in text)
    assert found, (
        f"missing TS literal {literal!r} in {TYPES_TS} — "
        f"docs/v2.1-daemon-ipc.md added it but types/daemon.ts did not mirror"
    )
```

`pre-Wave-4` skip lets the test land before agents create the file. Once agents create `app/src/types/daemon.ts`, the test goes green and the mirror is mechanically pinned forever after.

## 8. Frontend hooks

**`useDaemon`** — Context provider exposing daemon connection state.

```typescript
interface DaemonContextValue {
  status: "loading" | "running" | "stale" | "none" | "error";
  config: DaemonConfig | null;       // from get_daemon_config
  error: ShellError | null;
  startDaemon: (readonly: boolean, network?: XRPLNetwork) => Promise<void>;
  stopDaemon: () => Promise<void>;
  refresh: () => Promise<void>;
}
```

App.tsx wraps the router in `<DaemonProvider>`. Routes consume via `useDaemon()`.

**`useDaemonEvents`** — SSE subscription hook.

```typescript
function useDaemonEvents(handler: (event: SSEEvent) => void): void;
```

Opens `EventSource(`${config.url}/events`)` with token. Handler dispatches by `event.type`. Unsubscribes on unmount. Reconnects on disconnect with re-fetch (per Wave 3 spec §5 fire-and-forget contract).
