# Sovereignty v2.1 Roadmap

## Status

**Decisions locked 2026-05-01.** v2.1 is the App UI initiative + full engine generalization, run as a single coherent dogfood swarm post-v2.0.2 ship.

## Locked decisions

### App UI

| Question | Decision |
|---|---|
| Framework | **Tauri (v2)** — skill compounds with GlyphStudio + CommandUI; no new stack tax. Web UI inside means Claude artifacts mock screens directly. XRPL anchor chains visualize cleanly as HTML/SVG (Textual loses that). Electron + PyQt rejected for this workspace. |
| Scope | **Both views: audit viewer + game shell**, two views over the same engine. Same Rust shell, same web stack — second view is incremental, not 2× the work. A standalone audit viewer with no game shell is a half-product. |
| Distribution | **npm-launcher wrapper + GitHub Releases** (standalone Tauri binary). PyPI keeps the CLI; the desktop app does not bundle into PyPI. |
| Design tooling | **Claude artifacts for first mockups → hand-built React/Solid components** in the Tauri webview. No additional MCP. |

### Engine generalizations

Run as part of the same swarm, in this dependency order:

1. **LedgerTransport ABC** — architectural lever, **first wave**. Refactor the XRPL-shaped transport into an abstract base + an `XRPLTransport` concrete impl. Done before more code crystallizes around XRPL.
2. **`sov resume <game-id>` + multi-save model** — audit viewer requires it; persistence layer goes plural.
3. **Async transport sibling + daemon mode** — together. Daemon mode is what lets the Tauri app stay live without reloading state on every action; async sibling is the transport contract for it.
4. **Mainnet anchor support + multi-tx anchor consolidation** — together. Mainnet makes proof-verification real; multi-tx amortizes cost where cost matters.

### Deferred to v2.2

- **`sov undo` (state-history journal)** — multi-day refactor, no UI dependency, no story locked to it. The one cut that earns its place on the deferred list.

## Swarm wave plan

Single dogfood swarm on `mcp-tool-shop-org/sovereignty`. Sovereignty just shipped v2.0.2 with a full Health Pass clean, so v2.1 leads with Feature Pass on the new architectural pieces, then runs a fresh A/B/C/D Health Pass over the v2.1 deltas only, then Final + Treatment.

| Wave | Lens | What | Domains |
|---|---|---|---|
| 0 | Pre-flight | Save point tag, swarm init, control-plane domain map | Coordinator |
| 1 | Feature Execute | LedgerTransport ABC + XRPLTransport concrete impl | Backend (engine), Bridge (transport refactor), Tests, CI/Docs |
| 2 | Feature Execute | Multi-save model (`schema_version`-aware) + `sov resume <game-id>` | Backend (state model), Bridge (anchors.json plural index), Tests, CI/Docs |
| 3 | Feature Execute | Async transport sibling + daemon mode (IPC contract for Tauri) | Backend (daemon entry), Bridge (async transport), Tests, CI/Docs |
| 4 | Feature Execute | Mainnet anchor support + multi-tx anchor consolidation | Bridge (mainnet config + batching), Backend (cost model), Tests (incl. real-mainnet skip path), CI/Docs |
| 5 | Feature Execute | Tauri shell scaffold + IPC bindings to daemon | Tauri Shell (Rust), Bridge (IPC contract), Tests, CI/Docs |
| 6 | Feature Execute | Audit viewer + game shell views (React/Solid) | Web UI, Tauri Shell (window plumbing), Tests, CI/Docs |
| 7 | Health A | Bug/security audit on v2.1 deltas only | All 6 domains |
| 8 | Health B | Proactive audit on v2.1 deltas only | All 6 domains |
| 9 | Health C | Humanization amend (error messages, loading states, daemon disconnect feedback, audit-viewer empty states, accessibility) | Bridge, Web UI, CI/Docs primary |
| 10 | Health D | Visual polish (typography/spacing in webview, audit-chain SVG aesthetics, dark-mode parity, app icon, OG card refresh) | Web UI, Tauri Shell, CI/Docs primary |
| 11 | Final | Comprehensive test pass — Python pytest + Rust `cargo test` + web `vitest` + integration | Tests |
| 12 | Treatment | Shipcheck → version bump v2.1.0 → README + handbook updates (Tauri install, audit viewer page, daemon mode) → translations (Mike runs locally) → repo metadata → repo-knowledge DB → release | Coordinator + Treatment phases |

### Domain split (6 domains for v2.1, max 10 per protocol)

The standard 5-domain split (Backend / Bridge / Tests / CI-Docs / Frontend) gets adjusted because Tauri adds two distinct file-ownership layers:

| Domain | Files |
|---|---|
| Backend | `sov_engine/**` (Python — engine, state model, cost model, daemon entry) |
| Bridge | `sov_transport/**` (Python — LedgerTransport ABC, XRPLTransport, async sibling, anchors.json) |
| Tauri Shell | `app/src-tauri/**` (Rust — main, IPC commands, window plumbing, `Cargo.toml`, `tauri.conf.json`) |
| Web UI | `app/src/**` (React/Solid components, routes, styles) |
| Tests | `tests/**` (pytest), `app/src-tauri/tests/**` (Rust), `app/src/**/*.test.tsx` (vitest) |
| CI/Docs | `.github/workflows/**`, `*.md`, `docs/**`, `site/**`, `pyproject.toml`, `app/package.json` |

### Audit-lens carry-forward from v2.0.0

The v2.0.0 swarm post-mortem flagged one audit-lens improvement: cross-check workflow filenames against external auth records (PyPI Trusted Publishers, npm provenance) during the supply-chain audit pass. Apply this to Wave 7 (Stage A) and to Phase 12 Treatment shipcheck.
