# v2.1 Daemon + Async Transport — Contract Spec

**Status:** locked 2026-05-01, pre-Wave-3 dispatch.
**Run:** `swarm-1777686810-67fd`
**Resume from:** tag `swarm-wave-2-complete-1777686810` @ `66d4742`
**Owns:** Async XRPL transport sibling, daemon mode (HTTP/JSON localhost server), SSE event stream, daemon lifecycle, IPC contract that Tauri (Wave 4) and audit viewer (Wave 5) consume.

## Driver

**Tauri webview cannot tolerate per-click Python subprocess spawn.** ~200ms × N user actions = unusable UX. Daemon sits in memory, responds in <10ms. The audit viewer also wants background chain polling for "anchor confirmed" notifications without each tab spawning its own client.

Both consumers want HTTP/JSON over localhost. SSE for push updates. One daemon per project root.

## Scope

1. Async XRPL transport sibling (`AsyncXRPLTransport`, real async via `xrpl-py.AsyncJsonRpcClient`).
2. Shared internals extraction (`sov_transport/xrpl_internals.py`) — sync + async consume one source of truth.
3. Daemon HTTP server (Starlette + uvicorn, `[daemon]` opt-in extra).
4. Daemon HTTP surface — read endpoints (always-on), write endpoints (anchor only, gated by `--readonly`).
5. SSE event stream — fire-and-forget, no buffer.
6. Daemon lifecycle — foreground + detached background, pid-based start/stop/status.
7. CORS + bearer-token auth — origin `*`, token in `Authorization` header.
8. Seed-leak regression test — pinned in CI.

Out of scope (flagged for v2.2+):
- Gameplay verb endpoints (`/play`, `/upgrade`) — daemon serves audit + anchor only in v2.1.
- SSE event buffer + `Last-Event-ID` reconnect — fire-and-forget for v2.1; reconnecting clients re-fetch.
- Multi-daemon coordination across project roots (each project root owns one daemon).
- IPC versioning protocol (v2.1 ships v1; future-bump strategy is v2.2 work).
- Keychain seed integration (env-only in v2.1; v2.2 adds `--keychain` without IPC change).
- Auth methods beyond bearer token (TLS, mTLS, OAuth — none warranted for localhost).
- systemd/launchd integration.

## 1. Async transport sibling

`sov_transport/xrpl_async.py` (NEW) — `AsyncXRPLTransport` parallel to sync `XRPLTransport`:

```python
class AsyncXRPLTransport:
    def __init__(
        self,
        network: XRPLNetwork = XRPLNetwork.TESTNET,
        *,
        url: str | None = None,
        allow_insecure: bool = False,
    ) -> None: ...

    async def anchor_batch(self, rounds: list[BatchEntry], signer: str) -> str: ...
    async def is_anchored_on_chain(self, txid: str, expected_hash: str) -> bool: ...
    async def get_memo_text(self, txid: str) -> str | None: ...
    def explorer_tx_url(self, txid: str) -> str: ...   # sync, no I/O
```

**No `AsyncLedgerTransport` ABC.** Same logic as Wave 2's Signer-protocol skip: do not abstract without a second async impl pulling on it. `AsyncXRPLTransport` stands alone until a second async impl exists.

**xrpl-py async surface:** uses `xrpl.asyncio.clients.AsyncJsonRpcClient` and `xrpl.asyncio.transaction.submit_and_wait` (the async variants). Retry loop ports verbatim from sync impl with `await asyncio.sleep` substituted for `time.sleep`.

## 3. HTTP server framework

**Starlette + uvicorn**, behind a new `[daemon]` opt-in extra in `pyproject.toml`:

```toml
[project.optional-dependencies]
xrpl = ["xrpl-py>=2.0"]
daemon = ["starlette>=0.36", "uvicorn[standard]>=0.27", "httpx>=0.27"]
```

`httpx` listed for the test surface (`TestClient` async-friendly client). PyInstaller binaries bundle all extras; PyPI users opt in: `pip install 'sovereignty-game[xrpl,daemon]'`.

**Why not FastAPI:** ~10 endpoints, no Pydantic validation needed (request bodies are simple JSON, validated inline). Pydantic would be ~3MB of dep weight for no gain.

**Why not stdlib `wsgiref`:** SSE concurrent clients + background chain polling want `asyncio`. `wsgiref` is sync thread-per-client; SSE works but the polling background task gets awkward without an event loop.

## 5. SSE event types

Stream content-type `text/event-stream`. Each event: `event: <type>\ndata: <json>\n\n`. **No `id:` line** (no buffer / no `Last-Event-ID` support in v2.1).

| Event type | Payload | Emit trigger |
|---|---|---|
| `daemon.ready` | `{"network":"testnet","readonly":false,"ipc_version":1}` | First event on each new SSE connection |
| `daemon.shutdown` | `{"reason":"stop_command"}` | Daemon receives SIGTERM / `sov daemon stop` |
| `anchor.pending_added` | `{"game_id":"s42","round":"3","envelope_hash":"<hex>"}` | Helper `add_pending_anchor` triggers via daemon-side hook |
| `anchor.batch_complete` | `{"game_id":"s42","txid":"ABC...","rounds":["1","2","3","FINAL"],"explorer_url":"https://..."}` | After successful `anchor_batch` on this daemon's transport |
| `game.state_changed` | `{"game_id":"s42"}` | Polled mtime change on `.sov/games/<id>/state.json` |
| `error` | `{"level":"warn","code":"<code>","message":"<text>"}` | Daemon-level error worth surfacing |

**State-change detection:** daemon polls `.sov/games/*/state.json` mtimes every 1s when at least one SSE client is connected. Polling is simpler and cross-platform vs fsevents/inotify; 1s granularity is fine for human-perceptible UI. No polling when no SSE listeners (cost-free idle).

**Reconnect contract:** when an SSE client disconnects + reconnects, the new connection gets a fresh `daemon.ready` event and any subsequent ones. **Missed events between disconnect and reconnect are NOT replayed.** Clients re-fetch state via `GET /games/{id}` on reconnect.

## 6. `.sov/daemon.json` schema

Written atomically by daemon at start, removed on clean exit:

```json
{
  "schema_version": 1,
  "pid": 12345,
  "port": 47823,
  "token": "<base64-256-bit-random>",
  "network": "testnet",
  "readonly": false,
  "ipc_version": 1,
  "started_iso": "2026-05-01T18:23:11Z"
}
```

**Token generation:** `secrets.token_urlsafe(32)` → 256-bit URL-safe base64. Generated fresh at every daemon start. Never reused, never persisted beyond the lifetime of `.sov/daemon.json`.

**Port selection:** `socket.bind(('127.0.0.1', 0))` to claim a free port from the kernel, then bind uvicorn to it. Random port avoids collisions with other Sovereignty daemons in other project roots.

**Seed leak prevention:** the `XRPL_SEED` env value or `--signer-file` contents are NEVER written to this file. Pinned by the seed-leak regression test (§11).

**Atomic write + permissions:** written via `atomic_write_text(path, content, mode=0o600)` (owner-only). The `token` field is the daemon auth gate; world-readable token defeats the boundary. Stage A hardening; pre-Stage-A relied on default umask.

**Stage A field addition — `started_by_shell`:** the Tauri shell's `DaemonStatus` Rust struct (see `docs/v2.1-tauri-shell.md` §3) carries a `started_by_shell: bool` indicator that the frontend uses to enforce Settings guardrail #1 (refuse network switch on externally-started daemon). The flag is **shell-process-lifetime state, not persisted to `.sov/daemon.json`** — daemon trust boundary stays clean. Spec callout here so daemon-side reviewers don't get surprised by the shell's behavior.

## 6.1 Doctor coverage (Wave 7 Stage B)

`sov doctor` validates daemon presence WITHOUT making an HTTP call (doctor wall-time <2s; daemon may be slow-but-alive without being dead). Source of truth is the on-disk `.sov/daemon.json`:

| Check | Pass | Warn | Fail |
|---|---|---|---|
| `.sov/daemon.json` exists | yes |  | malformed JSON or `schema_version` unrecognized |
| pid alive (via `os.kill(pid, 0)`) | yes | dead pid (stale entry — operator should `sov daemon stop` to clean it up) |  |
| token format (32+ url-safe base64 chars) | yes |  | wrong shape |
| `schema_version` current | yes | older-but-supported | unrecognized |

Doctor does NOT attempt `GET /health` — health check stays in `sov daemon status` (which has its own connect-with-timeout flow). Pinned at spec level so the daemon-presence doctor check doesn't drift toward HTTP and blow the wall-time budget.

## 7. CORS posture

**`Access-Control-Allow-Origin: *`** on all endpoints. Tauri webview origin is `tauri://localhost` (macOS/Linux) or `http://tauri.localhost` (Windows) — cross-origin to `http://127.0.0.1:<port>`. The bearer token in `Authorization` header is the actual auth gate; CORS origin restrictions add no real security on a localhost-bound port.

**Other CORS headers:**
- `Access-Control-Allow-Methods: GET, POST, OPTIONS`
- `Access-Control-Allow-Headers: Authorization, Content-Type`
- `Access-Control-Max-Age: 600`

Preflight `OPTIONS` requests respond `204 No Content` with the headers above.

**Pinning rationale:** without explicit `*`, agents may default-restrict CORS to a specific origin and break Wave 4 (Tauri shell) silently. The CORS posture is locked at the contract level for exactly this reason.

## 8. Lifecycle

| Command | Behavior |
|---|---|
| `sov daemon` | Foreground. Runs uvicorn in current process. SIGINT (Ctrl-C) stops cleanly. Logs to stderr at `SOV_LOG_LEVEL`. **Test/dev mode.** |
| `sov daemon start` | Detached background. Spawns `python -m sov_daemon ...` via `subprocess.Popen(start_new_session=True)` on Mac/Linux, `creationflags=subprocess.DETACHED_PROCESS \| CREATE_NEW_PROCESS_GROUP` on Windows. Writes `.sov/daemon.json`. Returns immediately with port + token printed. |
| `sov daemon stop` | Reads `.sov/daemon.json`, sends `SIGTERM` (or `terminate()` on Windows). Polls `os.kill(pid, 0)` until exit (max 10s). Removes `.sov/daemon.json` on success. |
| `sov daemon status` | Reads `.sov/daemon.json` if present. Probes pid via `os.kill(pid, 0)`. Returns `running` / `stale` / `none`. |
| `sov daemon start --readonly` | Same as `start`, but no seed loaded. Anchor endpoints return 405. |
| `sov daemon start --network <network>` | Per-invocation network override (precedence #1 from Wave 2). |

**Stale state handling:**

- `sov daemon start` refuses if `.sov/daemon.json` exists AND pid is alive ("daemon already running on port X — `sov daemon stop` first").
- `sov daemon start` auto-cleans + proceeds if `.sov/daemon.json` exists AND pid is dead (stale entry from prior crash).
- `sov daemon status` distinguishes the three states explicitly.
- Daemon registers a `SIGTERM` / `SIGINT` handler that removes `.sov/daemon.json` on clean exit; crash recovery is the auto-clean path.

Locked because crash-recovery state handling is the kind of thing that gets right in normal flow and broken in edge cases.

See §6.1 for doctor-time validation, which is separate from `sov daemon status` runtime check (no HTTP call, <2s wall-time budget).

## 9. Wallet handling

Seed loaded **at daemon start**, never per-request:
- `XRPL_SEED` env var (default).
- `--signer-file <path>` flag (file contents, stripped).
- Held in memory only. Never logged. Never written to `.sov/daemon.json`.
- Readonly mode (`--readonly`) skips seed load entirely; anchor endpoints return 405 before any code path touches signing.

**Per-request seed is rejected** — it pushes seed handling into the Tauri webview JS path, larger attack surface, breaks the "daemon is the trust boundary" model.

## 10. Daemon scope discipline

**Daemon serves audit + anchor coordination only** in v2.1.

Gameplay verbs (`sov play`, `sov upgrade`, `sov vouch`, etc.) stay **CLI-only**. Tauri shell (Wave 4) and audit viewer (Wave 5) agents must NOT assume daemon endpoints exist for gameplay actions. The Tauri shell either:
- Spawns CLI subprocesses for gameplay actions (acceptable — game turns are not click-rate sensitive).
- Or the user plays via CLI in a separate terminal while Tauri shows the audit view.

Locked here so Wave 4/5 agents don't drift into building gameplay-over-IPC. v2.2+ extends if the UX demands it.

**Seed-leak test (the trust boundary pin):**

```python
def test_daemon_json_never_contains_seed(tmp_path, monkeypatch):
    """Mechanically pin: .sov/daemon.json never serializes the wallet seed.

    Trust boundary: daemon holds seed in memory only. Any serialization regression
    that lands the seed in the on-disk daemon.json must fail CI, not user testing.
    """
    secret = "sSecretSeed_" + "X" * 24
    monkeypatch.setenv("XRPL_SEED", secret)
    # Start daemon (mocked transport so no actual XRPL client init), inspect json
    state = start_daemon_and_read_state_file(tmp_path)
    assert secret not in state.read_text(encoding="utf-8")
    # Cover --signer-file path too
    seed_file = tmp_path / "seed.txt"
    seed_file.write_text(secret + "\n", encoding="utf-8")
    monkeypatch.delenv("XRPL_SEED")
    state2 = start_daemon_and_read_state_file(tmp_path, signer_file=seed_file)
    assert secret not in state2.read_text(encoding="utf-8")
```
