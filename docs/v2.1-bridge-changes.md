## Driver

**Audit ergonomics, not cost.** XRPL mainnet anchor cost is ~0.00001 XRP/tx, ≈$0.0002/game even at 30 rounds. Negligible — cost-driven batching doesn't earn architectural complexity. The thesis: **one verifiable chain pointer per game, not a scattered 16-tx trail.** Sovereignty's audit story breaks down if a verifier needs 16 tx URLs to reconstruct a game.

## Scope

1. XRPL network parameterization (testnet/mainnet/devnet → one class, one module).
2. Multi-tx anchor consolidation (game-end batch, multi-memo single-tx).
3. Verify contract split (local recompute vs chain lookup, 3-state status).
4. CLI `sov anchor` reshape (flush pending; deprecate proof-file form).
5. Network selection surface (env var > config > CLI flag).
6. Faucet rename (`fund_testnet_wallet` → `fund_dev_wallet(network=…)`).

Out of scope (flagged for v2.2): mainnet wallet seed mgmt beyond plain-file storage; LedgerTransport for non-XRPL chains; `sov undo`.

## 2. Multi-tx anchor consolidation

**Batch unit:** game-end is the natural flush point; mid-game flush is operator opt-in via `--checkpoint`.

**Memo format:** **multi-memo single-tx.** XRPL `Payment.Memos` accepts a list. One SOV grammar line per memo, N memos per Payment. **No grammar change** — existing format `SOV|<ruleset>|<game-id>|r<N>|sha256:<hash>` stays per-memo. Just N of them in the same Payment.

Why multi-memo, not single-memo packed:
- `MAX_ROUNDS=15` (`sov_engine/models.py:372`) + final = 16 hashes.
- Packed memo: ~30 (prefix) + 16 × 72 (entry) ≈ 1152 bytes — over the 1024 cap.
- XRPL allows ~150 memo fields per tx — multi-memo is forward-compatible with future high-round rulesets without split logic.

**Anchor flow:**

1. Round N completes → proof written to `.sov/games/<game-id>/proofs/round_<N>.proof.json`. Hash added to `.sov/games/<game-id>/pending-anchors.json` via `add_pending_anchor()`.
2. Game ends OR `sov anchor --checkpoint` → all pending hashes batched into one `Payment` with N memos. `transport.anchor_batch(rounds, signer)` submits.
3. On success: `clear_pending_anchors(game_id, round_keys)` clears flushed entries; `_record_anchor()` writes one entry per round into `anchors.json` keyed by round, all pointing at the same txid.
4. On failure: pending stays untouched. Operator retries with `sov anchor` (idempotent — re-batches the same pending set).

**New transport method** (parallel to existing single `anchor()`):

```python
def anchor_batch(
    self,
    rounds: list[BatchEntry],   # [(round_key, ruleset, game_id, envelope_hash), ...]
    signer: str,
) -> str:
    """Anchor N rounds in one Payment via N memos. Returns single txid."""
```

`BatchEntry` is a `TypedDict` or `dataclass` in `sov_transport/base.py` so all transports share the contract. See **Mirror discipline** below — `BatchEntry` is server-side only (engine→transport); no TS mirror in `app/src/types/daemon.ts` is required at v2.1. Existing `anchor(round_hash, memo, signer)` stays for backward compat with the deprecated single-round form; emits `DeprecationWarning` when called from `sov anchor <proof_file>` legacy path.

## 3. Verify contract split

Today's surface conflates two questions:
- "Does this proof file's `envelope_hash` match its contents?" (local, no chain hit)
- "Is this proof anchored on chain?" (chain lookup, no local check)

Batching forces splitting them, because in v2.1 a proof can exist locally with hash valid but chain entry **pending** (in `pending-anchors.json`, not yet flushed). Three states, not two:

```python
class AnchorStatus(StrEnum):
    ANCHORED = "anchored"   # txid present on chain, memo matches
    PENDING  = "pending"    # in pending-anchors.json, no txid yet
    MISSING  = "missing"    # not pending, no txid — never attempted or lost
```

**Transport surface stays pure-chain:**

```python
class LedgerTransport(ABC):
    @abstractmethod
    def is_anchored_on_chain(
        self, txid: str, expected_hash: str
    ) -> ChainLookupResult: ...
    # ChainLookupResult: StrEnum FOUND / NOT_FOUND / LOOKUP_FAILED (added Wave 6
    # Stage A — sov_transport/base.py). Engine collapses
    # FOUND → AnchorStatus.ANCHORED, NOT_FOUND / LOOKUP_FAILED → AnchorStatus.MISSING
    # at sov_engine/proof.py before any value crosses to the daemon/frontend.
    # Distinguishing transient lookup failure from definitive not-found prevents
    # the silent bool-collapse that masked transport errors pre-Stage-A.

    # Existing verify(txid, expected_hash) -> bool kept as compat alias,
    # returning is_anchored_on_chain(...) is ChainLookupResult.FOUND.
    # DeprecationWarning, removed v2.2.
```

### Mirror discipline — `ChainLookupResult` is intentionally Python-only

`ChainLookupResult` is **not** mirrored to TS. The engine collapses the 3-state to `AnchorStatus` (2-state from the frontend's perspective: anchored / pending / missing) at `sov_engine/proof.py::proof_anchor_status`, so the daemon's `/games/{id}/anchor-status/{round}` endpoint never emits FOUND / NOT_FOUND / LOOKUP_FAILED. The Python type-sync test (`tests/test_daemon_types_ts_in_sync.py`) deliberately omits `ChainLookupResult`. v2.2 may surface the 3-state if the audit viewer UX wants to distinguish transient failure from missing — at that point add the TS mirror and extend the parametrize set together. Until then: treat `ChainLookupResult` as an internal transport↔engine contract.

`BatchEntry` (the `TypedDict` in §2 above) is similarly Python-only by design. Frontend never constructs an anchor batch — anchor batching is the daemon's `POST /games/{id}/anchor` endpoint's responsibility. No TS mirror in `app/src/types/daemon.ts` is required at v2.1. If a v2.2+ feature surfaces per-round batch composition in the UI, extend the type-sync test at that point.

**Engine surface owns the 3-state composition** (consults pending index + transport):

```python
# sov_engine/proof.py
def verify_proof_local(proof_path: Path) -> bool:
    """Recompute envelope_hash from proof contents, compare to stored field."""

def proof_anchor_status(proof_path: Path, transport: LedgerTransport) -> AnchorStatus:
    """3-state: chain anchored | pending flush | missing. Reads pending-anchors.json
    via io_utils, then defers to transport.is_anchored_on_chain for ANCHORED check."""
```

CLI surfaces (`sov status --brief`, `sov doctor`) consume `proof_anchor_status` and render the 3-state.

## 4. `pending-anchors.json` schema

**Path:** `.sov/games/<game-id>/pending-anchors.json`. Atomic-written via `sov_engine/io_utils.py::atomic_write_text(path, content, mode=0o600)`. Owner-only — defensive even though no secret material lives in this file (defense-in-depth; Stage A hardening). Read sites consume `sov_engine/schemas.py::read_versioned` so unknown `schema_version` raises a structured error rather than silently misparsing.

```json
{
  "schema_version": 1,
  "entries": {
    "1":     { "envelope_hash": "<64-hex>", "added_iso": "<ISO8601-utc>" },
    "2":     { "envelope_hash": "<64-hex>", "added_iso": "<ISO8601-utc>" },
    "FINAL": { "envelope_hash": "<64-hex>", "added_iso": "<ISO8601-utc>" }
  }
}
```

Round keys match existing `anchors.json` convention: stringified round number `"1"`…`"15"` or `"FINAL"`.

**New helpers in `sov_engine/io_utils.py`** (alongside Wave 1's multi-save helpers):

| Helper | Purpose |
|---|---|
| `read_pending_anchors(game_id) -> dict[str, PendingEntry]` | Returns empty dict if file missing. |
| `add_pending_anchor(game_id, round_key, envelope_hash) -> None` | Idempotent: re-add same round_key overwrites timestamp only. |
| `clear_pending_anchors(game_id, round_keys: list[str]) -> None` | Partial clear, idempotent. Empty `round_keys` is a no-op. |
| `pending_anchors_path(game_id) -> Path` | Mirror of existing `proofs_dir()` etc. helpers. |

`PendingEntry` is a `TypedDict` exported alongside `GameSummary`.

### 4.1 `anchors.json` schema (Wave 7 Stage B)

**Path:** `.sov/games/<game-id>/proofs/anchors.json`. Wrapped shape:

```json
{
  "schema_version": 1,
  "anchors": {
    "1": "ABC...",
    "2": "DEF...",
    "FINAL": "GHI..."
  }
}
```

Round-key convention identical to `pending-anchors.json`: stringified round number `"1"`…`"15"` or `"FINAL"`, value is the XRPL txid for that round.

**Read tolerance:** the bare-dict shape from pre-v2.1 layouts (`{"1": "ABC...", "FINAL": "DEF..."}` with no envelope) is accepted on read and migrated-on-write to the wrapped shape. New writes always emit the wrapped form. Read goes through `sov_engine/schemas.py::read_versioned` once an envelope is detected; bare-dict reads fall through to a legacy decoder that synthesizes `schema_version=0` for log/observability purposes only.

**Atomic-write:** `atomic_write_text(path, content, mode=0o644)` (default — game-state mode, not token-bearing).

## 5. CLI changes

**`sov anchor`** behavior matrix:

| Invocation | Behavior |
|---|---|
| `sov anchor` (no args, game complete) | Flush all pending for active game. Single tx, N memos. |
| `sov anchor` (no args, game in progress) | Warn + suggest `--checkpoint`; require explicit confirm or fail with hint. |
| `sov anchor --checkpoint` | Mid-game flush, no warning. Same flush path. |
| `sov anchor --network <network>` | Per-invocation network override (precedence #1 above). |
| `sov anchor <proof_file>` | **Deprecated** — single-round legacy form. Emits `DeprecationWarning`, calls existing `anchor()`. Removed in v2.2. |
| `sov anchor` with empty pending | Idempotent no-op, `info` exit, no tx submitted. |

**`sov status --brief`** — extend per-round status field to 3-state via `proof_anchor_status`. Surface `pending_count` in the `--json` output schema (additive — does not bump `schema_version`).

**`sov doctor`** — new check under XRPL section: `pending_anchors`. The check covers operational freshness AND integrity:

| Pass | Warn | Fail |
|---|---|---|
| Zero pending OR all entries fresh (`added_iso` within 1 hour) AND `schema_version` recognized AND no orphan `round_keys` (every key references a `.proof.json`) | Pending > 0 AND oldest entry older than 1 hour (operator should run `sov anchor`) OR `schema_version` older-but-supported | `pending-anchors.json` malformed parse OR `schema_version` unrecognized OR `round_keys` reference rounds without `.proof.json` |

Schema-version validation goes through the centralized helper `sov_engine/schemas.py::read_versioned` (Wave 7 Stage B). Doctor MUST stay <2s wall-time on healthy systems; pending-anchors check is pure-fs-stat + JSON parse, no network call.

## 6. Faucet rename

`sov_transport/xrpl.py::fund_testnet_wallet()` → `fund_dev_wallet(network: XRPLNetwork = XRPLNetwork.TESTNET)`:

| Network | Behavior |
|---|---|
| `TESTNET` | Faucet auto-fund (current behavior). |
| `DEVNET` | Faucet auto-fund (devnet has its own faucet). |
| `MAINNET` | Raise `MainnetFaucetError` with operator-actionable message: `"mainnet has no faucet — set XRPL_SEED to a funded mainnet wallet, or run sov wallet --network testnet."` |

`fund_testnet_wallet()` stays as a thin compat shim, calls `fund_dev_wallet(TESTNET)`, emits `DeprecationWarning`. Removed in v2.2.

`MainnetFaucetError` is defined in `sov_transport/xrpl_internals.py` (Wave 3 internals extraction) and re-exported via `sov_transport/__init__.py`, code `MAINNET_FAUCET_REJECTED` in the structured-error registry.

## 7. Backward compatibility

Existing v2.0.2 testnet anchors are single-memo, single-round per tx. v2.1 verify path **must handle both shapes** — the existing `_extract_memos` iteration in `verify()` already does N≥1 memo iteration, so no code change is needed at the transport. Acceptance criteria: a v2.0.2-anchored proof file verifies green under v2.1's `is_anchored_on_chain`. Test: `tests/test_xrpl_transport_legacy.py` pins this with a recorded testnet tx fixture.

The CLI legacy form `sov anchor <proof_file>` keeps working (single-round path) so users with existing scripts don't break in v2.1; deprecation removal is v2.2.

## 8. Error codes added

| Code | Surface | Meaning |
|---|---|---|
| `MAINNET_FAUCET_REJECTED` | `fund_dev_wallet(MAINNET)` | Operator must provide funded mainnet seed. |
| `ANCHOR_PENDING` | `sov status` / `sov doctor` | Proof is queued in pending-anchors.json, not yet on chain. |
| `INVALID_NETWORK` | CLI / config | Network string not in `{testnet, mainnet, devnet}`. |
| `MAINNET_UNDERFUNDED` | `XRPLTransport.anchor_batch` on mainnet | Wallet balance < required reserve + fee. (Distinct from generic XRPL submission failure so operator gets the specific actionable hint.) |

All four follow the existing structured error shape (`code` / `message` / `hint`).
