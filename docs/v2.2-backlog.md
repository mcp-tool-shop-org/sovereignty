## Workspace-level desktop binary signing (Wave 11 deferral)

v2.1.0 ships Tauri shell binaries on GH Releases with **build-provenance attestation only** (`actions/attest-build-provenance`); first-launch OS warnings are expected and documented in the README. OS-level code signing was deferred from Wave 11 because it's workspace-level shared infrastructure that benefits sovereignty + GlyphStudio + CommandUI together — per-repo setup would triple the work for the same result.

**Concrete next-steps (workspace-level — owned by mcp-tool-shop, not sovereignty):**

1. **Apple Developer Program enrollment.** $99/yr. Annual renewal. Apple ID required.
2. **Apple Developer ID Application certificate generation.** Created via Xcode or Apple Developer portal. One cert covers all workspace Tauri apps.
3. **`notarytool` credentials configured.** Stored in 1Password (or equivalent secret manager) — App-Specific Password generated at appleid.apple.com, plus the team ID. CI consumes via `APPLE_ID`, `APPLE_PASSWORD`, `APPLE_TEAM_ID` repo / org secrets.
4. **Authenticode certificate acquisition.** EV preferred for SmartScreen reputation bypass (~$400-700/yr, ~1-2 week issuance through certificate authorities like DigiCert, Sectigo, GlobalSign). OV cert is cheaper (~$200-400/yr, hours-to-days issuance) but SmartScreen still warns until the binary builds reputation.
5. **Workflow templates updated to consume signing creds when present.** Modify `.github/workflows/publish.yml` Tauri matrix (current location: `sovereignty/.github/workflows/publish.yml::build-tauri-binaries`) to gate signing steps on `secrets.APPLE_ID != ''` etc. Templates copy to GlyphStudio + CommandUI on the same pattern.
6. **First-launch OS-warning README block removed** from sovereignty + GlyphStudio + CommandUI READMEs (and the Phase 8 playbook entry in `memory/full-treatment.md` updated to reflect signing-by-default).
7. **All workspace Tauri apps inherit signing once infrastructure lands.** Pair with Tauri 2 auto-updater wiring (also v2.2 work) — Tauri updater needs separate updater-signing keys, distinct from OS-level signing.

**Read order for v2.2 dispatch:** `memory/full-treatment.md` Phase 8 (the playbook), `memory/commandui-bootstrap.md` (workspace-level pattern reference — currently flags signing as "Windows/macOS gremlin work"), `docs/v2.1-bridge-changes.md` (sovereignty's distribution context).

**Memorialize in `memory/feedback_external_system_limits_must_be_validated.md` family if a fourth pattern surfaces:** "shared infrastructure deferred per-repo because cross-repo benefit beats sequential setup." Sovereignty Wave 11 is the precedent — not a regression, an explicit framing decision. Future shared-infrastructure deferrals should cite this precedent rather than re-litigating per-repo.
